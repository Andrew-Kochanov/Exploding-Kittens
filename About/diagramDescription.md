# Описание архитектуры

## Game

Ответственность: Агрегат игрового состояния. Хранит игроков, колоду, сброс, очередь хода, состояние, отложенное действие. Выполняет мутации состояния через свои методы, поддерживая целостность правил.

Смысл: Центральный узел игровой логики.

Инварианты:

- 2 ≤ число игроков ≤ 5.

- currentPlayerIndex всегда указывает на живого игрока.

- extraTurns ≥ 0.

- Количество «Взрывных котят» в колоде = (число живых игроков) − 1.

- В discardPile нет «Взрывных котят».

- state == WAITING_FOR_NOPE ⇔ pendingAction ≠ null.

## Player

Ответственность: Хранит идентификатор, имя, руку карт и статус жизни. Предоставляет операции с картами (добавление, извлечение).

Смысл: Проекция участника.

Инварианты:

- Живой игрок не может хранить «Взрывного котёнка» после завершения обработки взрыва.

- Мёртвый игрок имеет пустую руку.

- Все карты в руке соответствуют допустимым типам.

## Card

Ответственность: Неизменяемая карта (тип, имя).

Смысл: Value object.

Инварианты: Тип всегда из CardType, имя соответствует типу.

## Deck

Ответственность: Закрытая колода: перемешивание, взятие верхней карты, просмотр верхних, вставка в произвольное место для «Обезвредь».

Смысл: Инкапсулирует порядок карт.

Инварианты:

- Операции, кроме shuffle, сохраняют относительный порядок.

- peekTop не изменяет колоду.

- Вставка позиции от 1 (верх) до size+1 (низ).

## DiscardPile

Ответственность: Открытая стопка сброса, позволяет взять любую карту (для 5-карточной комбинации).

Смысл: Хранилище использованных карт.

Инварианты: Ни одной карты EXPLODING_KITTEN.

## CombinationEvaluator (object)

Ответственность: Определение, является ли набор карт комбинацией (2/3 одинаковые, 5 разных), и возврат эффекта.

Смысл: Чистая функция без состояния.

Инварианты: Возвращает корректный CombinationEffect или null, не модифицирует данные.

## Иерархия Rule

Ответственность интерфейса: Контракт проверки одного правила.

Смысл: Гибкое расширение валидации.

- PlayCardRule: можно ли играть карты (ход, наличие, состояние).

- DrawCardRule: можно ли взять карту (не завершён ли ход атакой/скипом).

- NopeRuleValidator: допустимость «Неть» на целевое действие.

- AttackSkipRule: корректность Attack/Skip и счётчика ходов.

- CombinationRule: действительность комбинации и цели кражи.

- ExtraTurnRule: обязательность дополнительных ходов.

## RuleSet

Ответственность: Содержит упорядоченный список правил, последовательно применяет их к действию.
 
Смысл: Координатор валидации.

Инварианты: Порядок правил фиксирован, validate не меняет игру, возвращает список ошибок.

## Иерархия Action (sealed class)

Ответственность: Типобезопасные команды от игроков.

Смысл: Исключает противоречивые состояния.

Инварианты:

- PlayCardsAction: карты не пусты, принадлежат игроку; комбинация валидна, если задана.

- DrawAction: игрок в фазе взятия.

- UseNopeAction: цель – существующее отложенное действие.

- DefuseInsertAction: позиция в 1..(deck.size+1).

- StealRandom/StealSpecific: цель жива.

- ResolveAction: содержит исходное отложенное действие.

## ActionResult (sealed class)

Ответственность: Сообщение о результате выполнения действия. 

Смысл: Обеспечивает однонаправленный поток. 

Инваарианты:

- ActionSuccess(events)

- ActionInvalid(errors)

- ActionAwaitingNope(pendingAction)

## GameSession

Ответственность: Управляет одной партией, связывая Game, RuleSet и лог ходов. Выполняет действия, возвращает ActionResult. 

Смысл: Фасад для UI. 

Инварианты:

- В состоянии WAITING_FOR_NOPE принимаются только UseNopeAction или ResolveAction.

- Лог соответствует выполненным действиям.

- pendingAction синхронизирован с game.pendingAction.

## GameSessionManager

Ответственность: Жизненный цикл сессий (создание, загрузка, сохранение, выход, завершение приложения). Координирует сервисы. Предоставляет список известных игроков для UI. 

Смысл: Единая точка входа для UI. 

Инварианты:

- Не более одной активной сессии.

- При создании новой игры предыдущая корректно завершается.

- При выходе из приложения данные сохраняются.

- getKnownPlayerNames() возвращает актуальный список из реестра.

## PlayerRegistry

Ответственность: Хранит множество имён всех игроков, когда-либо участвовавших в партиях. Предоставляет методы для получения списка имён и регистрации новых. 

Смысл: Позволяет выбирать игроков из существующих при создании новой игры, избегая дублирования и перемешивания статистики. 

Инварианты:

- Имена уникальны без учёта регистра (или с учётом — по выбранной политике).

- Список имён синхронизирован с CSV-хранилищем.

- При добавлении уже существующего имени ничего не происходит.

- registerAll гарантирует, что все переданные имена появятся в системе.

## CsvPlayerRepository

Ответственность: Низкоуровневая работа с CSV-файлом, содержащим имена игроков (по одному на строку). 

Смысл: Изолирует формат хранения реестра. 

Инварианты: Файл всегда содержит актуальный набор уникальных имён.

## StatisticsService

Ответственность: Пересчёт статистики (победы, поражения, обезвреживания и т.п.) по итогам партии, используя CsvStatisticsRepository для хранения. 

Смысл: Аналитика без привязки к UI. 

Инварианты: После updateStats CSV-файл статистики обновлён.

## CsvStatisticsRepository

Ответственность: Чтение/запись агрегированной статистики в CSV-файл. 

Смысл: Сменяемое хранилище для статистики. 

Инварианты: Файл корректен, записи соответствуют структуре PlayerStats.

## HistoryService

Ответственность: Сохранение протоколов завершённых партий и предоставление истории. 

Смысл: Аудит игр. 

Инварианты: Сохранённые партии полны, запросы возвращают корректные данные.

## TurnRecord

Ответственность: Данные одного хода. 

Смысл: Единица лога.

## GameView (интерфейс)

Ответственность: Контракт для UI: отображение игры, запрос действий, уведомления.

Смысл: Абстрагирует конкретный интерфейс.

## ConsoleUI

Ответственность: Реализация текстового интерфейса. При создании игры использует getKnownPlayerNames() для помощи пользователю. 

Смысл: Первый рабочий UI. 

Инварианты: Корректно обрабатывает все виды ActionResult, управляет циклом опроса «Неть».

## Main

Ответственность: Точка входа, инициализация компонентов и запуск ConsoleUI
